# 🚀 Referência Rápida - Agentes e Skills

## 📌 Comandos Mais Usados

### Debugging

```
"Debugger, encontre por que [problema]"
"Use systematic-debugging para [bug]"
```

### Desenvolvimento

```
"Frontend Specialist, crie [componente]"
"Backend Specialist, implemente [API]"
```

### Otimização

```
"Performance Optimizer, otimize [arquivo]"
"Use clean-code para refatorar [código]"
```

### Testes

```
"Test Engineer, crie testes para [funcionalidade]"
"Execute workflow test"
```

### Documentação

```
"Documentation Writer, documente [sistema]"
```

## 💡 Templates de Comandos

### Bug Fixing

```
"Debugger, [descrição do bug]. Use systematic-debugging 
para encontrar a causa raiz e corrija"
```

### Nova Feature

```
"Orchestrator, implemente [feature]. Use workflow plan 
primeiro, depois coordene os agentes necessários"
```

### Refatoração

```
"Use clean-code para refatorar [arquivo], melhorando 
legibilidade e mantendo funcionalidade"
```

### Performance

```
"Performance Optimizer, [arquivo] está lento. 
Aplique performance-profiling e otimize"
```

### Testes

```
"Test Engineer, crie testes completos para [módulo] 
usando testing-patterns"
```

## 🔥 Atalhos Rápidos

| Situação | Comando Rápido |
|----------|----------------|
